CASE STUDY — FRAME NO. 02

Forte Rewards: one loyalty program, four restaurants, zero identity crisis.

A cross-property loyalty system built to feel native to each restaurant while running on one shared points engine underneath.

Client
Forte Hospitality Group
Role
Program design & documentation
Scope
Loyalty strategy, Toast configuration, guest comms
Platform
Toast Loyalty

1920 × 1080

01 — The brief

Four concepts, one guest, no shared program

Forte Hospitality Group runs four distinct restaurants — Sapori Trattoria, Butcher & The Bear, Ithaki Estiatorio, and Ethos Taverna — each with its own menu, price point, and personality. Regulars who visited more than one concept had no way to be recognized as the same guest, and there was no structured incentive to bring them back or to move them between properties.

The ask: build a loyalty program that works across all four restaurants without flattening any of them into a single generic brand.

1600×1200

02 — The approach

Design inside the platform's real constraints

Toast Loyalty was already the point-of-sale layer at every location, so the program needed to be built natively inside it rather than bolted on with a third-party app — that meant designing around what Toast's tier, point-accrual, and redemption settings could actually support, not an idealized system on paper.

  • Mapped a three-tier structure (entry, mid, top) based on realistic annual spend bands across the group, not a single restaurant's price point
  • Kept each restaurant's own branding on guest-facing materials, with Forte Rewards as the connective layer rather than a replacement identity
  • Wrote the configuration as a step-by-step checklist so any location's manager could set it up correctly without a Toast support call
1600×1200

03 — The solution

A tier system guests actually understand

Three tiers, each with clear, spend-based thresholds and a visible reason to reach the next one. Perks were kept simple and operationally realistic — nothing that required special-casing at the register — so every location could deliver on the promise consistently.

Documentation shipped in three parts: full program rules for management, a Toast configuration checklist for setup and QA, and a one-page guest-facing explainer designed to be handed out or posted at the host stand.

04 — The result

One rollout, four locations, no retraining fire drill

Because the configuration checklist did the heavy lifting, every location could enroll and launch on the same timeline without a dedicated training session — managers followed the checklist, cross-checked against the guest-facing one-pager, and were live.

The bigger shift was structural: Forte now has a single view of guest loyalty across four previously disconnected concepts, with room to layer in cross-property promotions later without redesigning the whole system.
